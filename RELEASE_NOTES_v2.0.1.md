# CS-Scout v2.0.1 发布说明

这是面向 Windows 玩家版的兼容性补丁，主要解决远程云电脑或其他软件占用 TCP 5000
端口时无法启动的问题。现有密钥、Demo 缓存和分析输出都会继续保留。

## 修复内容

- Windows 玩家版不再固定使用 5000 端口，也不会要求结束占用端口的未知程序。
- 启动时由 Windows 内核原子分配空闲的本地端口，避免“先探测、后绑定”的竞态。
- Python 子进程通过一次性随机令牌、进程 ID 和原子 JSON 文件回报实际端口；启动器验证后
  才进行健康检查、鉴权检查并打开浏览器。
- 服务仍然只监听 `127.0.0.1`，不会开放到局域网或互联网，也不需要管理员权限或防火墙规则。
- 改进受限云电脑上的密钥 ACL 处理：不再尝试更改文件所有者；若底层文件系统仍不允许
  收紧 ACL，只显示非致命警告。
- 低磁盘提醒现在会直接建议先使用“普通”模式和 1–2 个 Demo。

## 下载与安装

下载并完整解压：

```text
CS-Scout-Windows-x64-v2.0.1.zip
```

然后依次双击：

1. `windows\Install-CS-Scout.cmd`
2. `windows\Start-CS-Scout.cmd`

不要直接在 ZIP 预览窗口中运行，也不要使用管理员身份启动。启动窗口会显示本次实际使用的
`Address`；端口数字每次不同属于正常现象。

## 升级说明

可以把 v2.0.1 解压到新目录，再运行安装脚本。数据仍保存在：

```text
%LOCALAPPDATA%\CS-Scout
```

因此访问密钥、Demo 缓存和分析输出不会因更换程序目录而丢失。确认新版本可用后，可以删除
旧的 v2.0.0 解压目录。

## 磁盘提醒

12–16 GB 可用空间仍能启动，但不适合一次分析很多玩家和 Demo。建议先使用“普通”模式、
1 名玩家和 1–2 个 Demo；停止 CS-Scout 后可删除 `%LOCALAPPDATA%\CS-Scout\demos`
释放缓存空间。

## 完整性校验

下载 ZIP 与 `SHA256SUMS.txt` 后，在 ZIP 所在目录运行：

```powershell
Get-FileHash -Algorithm SHA256 .\CS-Scout-Windows-x64-v2.0.1.zip
```

结果必须与校验文件中的 SHA-256 完全一致。
