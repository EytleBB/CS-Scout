# CS-Scout v2.0.0 发布说明

CS-Scout 2.0 是面向 Counter-Strike 2 Demo 的多玩家、多回合回放分析版本。本次发布同时提供 Windows x64 玩家包；普通玩家不需要部署服务器，也不需要开放公网端口。

项目原创源代码自本版本起采用 MIT License；第三方组件、商标与素材边界见
`THIRD_PARTY_NOTICES.md`。

## 下载与安装

请从项目官方 GitHub Releases 页面下载专用文件：

```text
CS-Scout-Windows-x64-v2.0.0.zip
```

不要下载 GitHub 自动生成的 `Source code (zip)` 或 `Source code (tar.gz)` 代替 Windows 玩家包；自动源码包可能不包含雷达地图和完整启动文件。

运行要求：

- Windows 10 或 Windows 11，64 位；
- 64 位 Python 3.11 或 Python 3.12，并保留 Python Launcher；
- 安装依赖、查找比赛和下载 Demo 时可以访问互联网；
- 建议至少保留 16 GB 可用磁盘空间，较深的分析会保存更多 Demo；
- 快速模式会显著增加 CPU、内存和网络占用。

安装步骤：

1. 先按下一节校验 ZIP 的 SHA-256。
2. 右键 ZIP 并选择“全部解压”，不要直接在压缩包预览窗口内运行。
3. 普通双击 `windows\Install-CS-Scout.cmd`；不要选择“以管理员身份运行”。
4. 首次安装会在发布包中创建独立的 `.venv` 并下载固定版本的 Python 运行依赖。
5. 安装完成后双击 `windows\Start-CS-Scout.cmd`。
6. 浏览器会打开 <http://127.0.0.1:5000>，本机访问密钥会复制到剪贴板；在网页密钥框中按 `Ctrl+V`。
7. 第一次测试建议选择“普通”模式，只填写一名玩家并分析一个 Demo。

完整使用和故障排查说明见 `windows\README-PLAYER-ZH.md`。

## 校验下载文件

Release 页面应同时提供 `SHA256SUMS.txt`。在 ZIP 所在目录打开 PowerShell，执行：

```powershell
Get-FileHash -Algorithm SHA256 .\CS-Scout-Windows-x64-v2.0.0.zip
```

将输出的 64 位十六进制散列值与 `SHA256SUMS.txt` 中对应文件的值逐字比较。若不一致，不要运行该文件；请删除后从项目官方 Release 页面重新下载。SHA-256 可以检查文件是否与发布者提供的文件一致，但仍需确认下载页面属于项目官方 GitHub 仓库。

## 主要功能

- 支持 Ancient、Anubis、Dust2、Inferno、Mirage、Nuke、Overpass 和 Train 共 8 张地图；
- 一次可填写最多 5 名玩家，每名玩家可选择分析 1–10 个 Demo；
- 自动查找 5E 玩家、比赛和可下载 Demo，并缓存及去重已下载的比赛；
- 解析回合冻结结束后的 20 秒走位、投掷物轨迹、落点和持续范围；
- 将所有成功加载玩家的手枪局合并显示，购买局则按玩家按钮分别查看；
- 统一的 CT/T 选择器，以及 1x、2x、4x 播放速度，默认 2x；
- 展示 K/D、AWP 持有率、回合数，并显示投掷物飞行图标；
- 雷达画布会根据窗口高度缩放，避免为查看完整地图而持续向下滚动。

## 普通与快速模式

| 模式 | 执行方式 | 适合场景 |
| --- | --- | --- |
| 普通 | 保留原有稳定的生产者/消费者流水线，在下载后按稳定顺序解析 | 首次使用、配置较低、希望减少资源占用或排查问题 |
| 快速 | 并行查找多名玩家，同时下载多个 Demo，并用多个进程重叠执行 Demo 解析 | CPU、内存和带宽充足，希望缩短多玩家或多 Demo 的总耗时 |

普通模式仍是默认选项，没有被快速模式替代。快速模式分别限制下载线程和解析进程；同一比赛被多名玩家引用时会复用一次下载。最终玩家顺序、Demo 顺序和回合编号在并发完成顺序不同的情况下仍按输入及源 Demo 顺序组装。

快速模式的目标是提高总体吞吐量，不保证每次任务都按固定倍数加速。第三方服务器响应速度、磁盘性能、Demo 大小、CPU 核数和可用内存都会影响结果。运行快速模式时电脑变慢、风扇加速或网络占用升高属于可能出现的资源负载；需要继续进行其他工作时请使用普通模式或降低分析深度。

## 本地数据与安全边界

Windows 启动器固定只监听 `127.0.0.1:5000`，仅本机可以连接。无需也不应开放 Windows 防火墙端口、路由器端口映射或云服务器安全组。这个 Windows 玩家包不是可直接暴露到公网的生产服务器配置。

每台电脑会生成自己的随机访问密钥。请勿把密钥粘贴给陌生人，也不要上传 `secret.key`。启动器不会要求管理员权限，不会修改系统防火墙，也不会主动结束与本次 CS-Scout 启动进程无关的程序。

持久数据保存在：

```text
%LOCALAPPDATA%\CS-Scout\
├─ secret.key
├─ demos\
└─ output\
```

Demo 和输出可能包含玩家用户名、SteamID、比赛记录、位置和战斗数据。公开分享前请确认有权处理并按需要脱敏；不要把密钥、Demo 或未脱敏输出附到公开 Issue。更多第三方、商标、平台条款与隐私说明见 `THIRD_PARTY_NOTICES.md`。

分析运行期间不要关闭启动窗口或删除 Demo 缓存。没有任务时，可以关闭窗口或按 `Ctrl+C` 停止服务；需要释放空间时，停止服务后再清理 `%LOCALAPPDATA%\CS-Scout\demos`。

## 从旧 v0.1 升级

v2.0 的服务结构、网页回放和输出格式与旧 v0.1 不同，不支持在旧目录中直接覆盖安装：

1. 等待旧版任务结束并停止旧版程序。
2. 备份旧版配置、Demo 和结果；不要先删除旧目录。
3. 将 v2.0.0 Windows ZIP “全部解压”到一个全新的目录。
4. 不要复制旧版 `.venv`、Python 包、启动脚本、密钥配置或生成的 JSON 到新目录。
5. 运行新版 `windows\Install-CS-Scout.cmd`，再用普通模式完成“一名玩家、一个 Demo”的测试。
6. 确认新版可正常启动、分析和回放后，再自行决定是否删除旧版程序目录。

旧 v0.1 的本地数据不会自动迁移或删除。v2.0 开始使用 `%LOCALAPPDATA%\CS-Scout` 保存密钥、Demo 缓存和输出；以后更换 v2 系列发布包时，这些数据会继续保留。若旧版也曾使用同名目录，请先完整备份，再安装 v2.0.0。

## 已知边界

- 分析依赖 5E 页面、接口及 Demo 下载地址；网络故障、平台变化或目标玩家没有可下载比赛时，可能没有结果；
- 当前只支持上述 8 张地图；
- 同一个本地实例一次只运行一个分析任务；
- Demo 文件较大，长时间、多玩家和高深度分析会占用明显的网络、CPU、内存和磁盘；
- 这是分析与回放工具，生成结果应结合 Demo 完整性和实际比赛情况判断。
